## Agile Development Evidence

### TDD Evidence
- [ ] Unit test files exist
- [ ] Tests written before implementation (show git history)
- [ ] Test coverage report generated
- [ ] CI runs tests on push

### Code Review Evidence
- [ ] All merges go through pull requests
- [ ] Each PR has at least 1 reviewer approval
- [ ] PR comments show constructive review

### Pair Programming Evidence
- [ ] Sessions logged with date, pair, duration, topic
- [ ] At least 3 sessions documented

### CI/CD Evidence
- [ ] CI workflow file exists
- [ ] Build runs automatically on push
- [ ] Tests run as part of CI
- [ ] Build artefact produced
